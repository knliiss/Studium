# Frontend Screen API Map

Use this document together with [API_CONTRACT.md](/Users/knalis/Documents/dev/java/Studium/docs/API_CONTRACT.md) and [FRONTEND_API_START.md](/Users/knalis/Documents/dev/java/Studium/docs/FRONTEND_API_START.md).
All frontend traffic must go through the gateway at `http://localhost:8080`.

## Auth

| Screen | Route suggestion | Required role | Main endpoint(s) | Request params/body | Response DTO | Notes |
| --- | --- | --- | --- | --- | --- | --- |
| Login | `/login` | public | `POST /api/auth/login` | `LoginRequest` | `AuthResponse` | Username-based login is the current local baseline. |
| Register | `/register` | public | `POST /api/auth/register` | `RegisterRequest` | `AuthResponse` | Can lead directly into authenticated app shell. |
| MFA challenge | `/mfa` | public challenge flow | `POST /api/auth/mfa/challenges/dispatch`<br>`POST /api/auth/mfa/challenges/verify`<br>`GET /api/auth/mfa/challenges/status` | `MfaDispatchChallengeRequest`<br>`MfaVerifyChallengeRequest` | `MfaChallengeResponse`<br>`AuthResponse` | Needed only when login response indicates MFA challenge state. |
| Password reset | `/password-reset` | public | `POST /api/auth/password-reset/request`<br>`POST /api/auth/password-reset/confirm` | `RequestPasswordResetRequest`<br>`ConfirmPasswordResetRequest` | `AcceptedActionResponse`<br>empty body | Current contract supports request and confirm only. |

## Student

| Screen | Route suggestion | Required role | Main endpoint(s) | Request params/body | Response DTO | Notes |
| --- | --- | --- | --- | --- | --- | --- |
| Student dashboard | `/student/dashboard` | `STUDENT` | `GET /api/v1/dashboard/student/me` | none | `StudentDashboardResponse` | Aggregated screen for schedule, deadlines, grades, tests, notifications, and analytics summary. |
| My schedule | `/student/schedule` | `STUDENT` | `GET /api/v1/schedule/me/week`<br>`GET /api/v1/schedule/me/range`<br>`GET /api/v1/schedule/me/export.ics` | `startDate` or `dateFrom`/`dateTo` | `ResolvedLessonResponse[]` or `text/calendar` | Default student schedule entrypoint should use BFF `/me` endpoints. |
| Schedule search | `/student/schedule/search` | `STUDENT` | `GET /api/v1/schedule/search` | `groupId?`, `teacherId?`, `roomId?`, `lessonType?`, `dateFrom`, `dateTo` | `ResolvedLessonResponse[]` | Shared read flow across all authenticated roles. |
| Subjects list | `/student/subjects` | `STUDENT` | `GET /api/v1/education/groups/by-user/{userId}`<br>`GET /api/v1/education/subjects/group/{groupId}` | path `userId`, then path `groupId`, plus pagination | `GroupMembershipResponse[]`<br>`SubjectPageResponse` | Frontend should resolve the current student's group membership first. |
| Subject details | `/student/subjects/:subjectId` | `STUDENT` | `GET /api/v1/education/subjects/{id}`<br>`GET /api/v1/education/topics/subject/{subjectId}` | path `subjectId`, plus pagination | `SubjectResponse`<br>`TopicPageResponse` | Use topics list to render the subject outline. |
| Topic details | `/student/topics/:topicId` | `STUDENT` | `GET /api/v1/education/topics/subject/{subjectId}`<br>`GET /api/v1/assignments/topic/{topicId}`<br>`GET /api/v1/testing/tests/topic/{topicId}` | path `subjectId` or `topicId`, plus pagination | `TopicPageResponse`<br>`AssignmentPageResponse`<br>`TestPageResponse` | The current contract exposes topics through the subject-scoped list only. Selected topic detail should be derived from `TopicPageResponse` items. |
| Assignment details | `/student/assignments/:assignmentId` | `STUDENT` | `GET /api/v1/assignments/{id}`<br>`GET /api/v1/submissions/{submissionId}/comments` | path ids | `AssignmentResponse`<br>`SubmissionCommentResponse[]` | Comments are submission-scoped, not assignment-scoped. |
| Submit assignment | `/student/assignments/:assignmentId/submit` | `STUDENT` | `POST /api/files`<br>`POST /api/v1/submissions` | multipart file + `fileKind`<br>`CreateSubmissionRequest` | `StoredFileResponse`<br>`SubmissionResponse` | Submit flow normally uploads the file first and then sends `assignmentId` + `fileId`. |
| My submissions | `/student/submissions` | `STUDENT` | `GET /api/v1/dashboard/student/me`<br>`GET /api/v1/assignments/{id}` | none, then assignment ids from dashboard cards | `StudentDashboardResponse`<br>`AssignmentResponse` | The current contract does not expose a dedicated personal submissions list endpoint. |
| Test list | `/student/tests` | `STUDENT` | `GET /api/v1/testing/tests/topic/{topicId}` | path `topicId`, pagination | `TestPageResponse` | Topic-driven list. |
| Test passing | `/student/tests/:testId` | `STUDENT` | `POST /api/v1/testing/tests/{id}/start`<br>`POST /api/v1/testing/results` | path `testId`<br>`CreateTestResultRequest` | empty body<br>`TestResultResponse` | The current public flow is start attempt, then submit result. There is no separate public test-by-id read endpoint. |
| My grades | `/student/grades` | `STUDENT` | `GET /api/v1/dashboard/student/me`<br>`GET /api/v1/analytics/students/{userId}` | path `userId` | `StudentDashboardResponse`<br>`StudentAnalyticsResponse` | No dedicated public grade list endpoint exists yet. Current UI should rely on dashboard and analytics summaries. |
| Notifications | `/student/notifications` | `STUDENT` | `GET /api/notifications/me`<br>`GET /api/notifications/me/unread-count`<br>`PATCH /api/notifications/{notificationId}/read`<br>`PATCH /api/notifications/me/read-all` | pagination, optional `status`, path `notificationId` | `NotificationPageResponse`<br>`UnreadCountResponse`<br>`NotificationResponse` | Personal notification center. |
| Profile | `/student/profile` | `STUDENT` | `GET /api/profile/me`<br>`PATCH /api/profile/me`<br>`PUT /api/profile/me/avatar`<br>`DELETE /api/profile/me/avatar` | `UpdateMyProfileRequest`<br>`UpdateAvatarRequest` | `UserProfileResponse` | Avatar update requires an existing `fileId` with kind `AVATAR`. |
| My analytics | `/student/analytics` | `STUDENT` | `GET /api/v1/analytics/students/{userId}`<br>`GET /api/v1/analytics/students/{userId}/subjects`<br>`GET /api/v1/analytics/students/{userId}/risk` | path `userId` | `StudentAnalyticsResponse`<br>`SubjectAnalyticsResponse[]`<br>`StudentRiskResponse` | Non-admin student access is self-scoped. |
